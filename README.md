## 功能概览

1. 自动注册 Free 账号
2. PayPal 激活 Plus 全流程
3. Hotmail / Outlook 自动别名
4. PayPal 接码号码池管理
5. 自动 OAuth 回调到本地及各类面板
6. 支持跳过 OAuth，导出只带 AT 的 JSON
7. 支持自建 Cloudflare Temp Email / Cloud Mail
8. 支持 DuckDuckGo Email Protection、iCloud Hide My Email 等邮箱链路

## 使用前须知

> [!IMPORTANT]
> 目前 OAuth 风控依然很重，很多场景都会跳手机号绑定。
>
> **更推荐使用：`导出至 - SESSION JSON 导入`**。

你至少需要准备这些东西：

1. 1 个稳定可用、能连续接收验证码的 US `+1` 接码手机号
2. 1 个或多个可用邮箱，或你自己的邮件系统
3. 1 条相对干净的 US 代理
4. 如果涉及支付转换，可能还需要 JP 出口代理或云端转换方案

> [!NOTE]
> 自建 Cloudflare Temp Email / Cloud Mail 若想争取试用资格，通常要使用 `edu` 前缀域名，例如 `edu.openai.com`。

## 安装教程

### 方式一：直接下载 Release 压缩包

1. 打开本仓库的 Releases 页面（发布后为 `LImingcheng07/gujumpgate-community`）
2. 下载最新的扩展压缩包
3. 解压到一个你方便管理的本地目录
4. 打开 `chrome://extensions/`
5. 打开右上角“开发者模式”
6. 点击“加载已解压的扩展程序”
7. 选择刚才解压出来的目录
8. 打开扩展详情页，勾选“允许在无痕模式下运行”
9. 如果你更新过旧版本，记得点一次“重新加载”

### 方式二：直接从源码目录加载

1. 下载或克隆本仓库源码
2. 不要把个人配置文件直接提交回仓库
3. 打开 `chrome://extensions/`
4. 开启开发者模式
5. 点击“加载已解压的扩展程序”
6. 选择仓库根目录
7. 完成后打开侧边栏开始配置

## 详细使用教程

### 第 1 步：启动扩展

安装完成后，先确认扩展能正常显示侧边栏。如果浏览器里加载后没有看到最新改动，请手动在扩展页点击一次“重新加载”。

### 第 2 步：启动本地 Helper

无论你是不是使用 Hotmail / Outlook，只要你的导出链路依赖本地 helper，都建议先启动：

- Windows：`start-hotmail-helper.bat`
- macOS：`start-hotmail-helper.command`

如果 helper 没有启动，很多本地导出、验证码读取、JSON 保存相关能力都可能失败。

### 第 3 步：准备代理

推荐思路通常仍然是：

- **US 注册**
- **JP 处理部分支付转换链路**
- **US 完成付款**

你可以按自己的环境选择：

#### 方案 A：云端转换

如果你已经有可用的云端支付转换方案，优先走云端，省去本地分流配置的复杂度。

#### 方案 B：本地代理转换

如果你使用本地代理转换：

1. 给注册链路准备 US 代理
2. 给支付转换链路准备 JP 代理
3. 在扩展侧边栏中按需填写转换代理 URL
4. 有测试按钮的地方先点测试，确认代理是通的再开始跑流程

### 第 4 步：配置邮箱渠道

你可以按自己的环境选择邮箱来源，例如：

- Outlook / Hotmail
- Cloudflare Temp Email
- Cloud Mail
- DuckDuckGo Email Protection
- iCloud Hide My Email
- 自定义邮箱池

如果你使用自建邮件方案：

1. 先把基础地址、管理认证、接收邮箱这些信息准备好
2. 在侧边栏对应位置填写
3. 点保存
4. 能测试的链路先测试，避免跑到一半才发现邮箱不可用

### 第 5 步：选择导出策略

目前更推荐：

- `导出至 - SESSION JSON 导入`

说明：

- `OAuth`：带刷新令牌，但现在风控通常更重
- `SESSION`：不带刷新令牌，但在很多当前环境里更实用

### 第 6 步：填写验证码接口 / 接码手机号 / 其他外部服务

你需要按自己的环境填写：

- 验证码接口 URL
- PayPal 接码手机号
- 代理配置
- 第三方面板地址
- 本地或云端导出目标

这些配置都属于**个人环境配置**，**不要直接提交到公开仓库**。

### 第 7 步：开始运行

保存配置后再开始跑流程。建议第一次先手动单次运行，不要一上来就批量自动化。

推荐排查顺序：

1. 先确认邮箱链路可用
2. 再确认验证码链路可用
3. 再确认代理转换链路可用
4. 最后再跑完整注册与支付流程

## 更新扩展

如果你已经在本地安装过旧版本：

1. 下载新版本文件
2. 覆盖你本地的扩展目录
3. 打开 `chrome://extensions/`
4. 对这个扩展点一次“重新加载”

如果不重新加载，浏览器可能还在继续使用旧版本代码。

## 公开仓库不包含什么

为了避免把维护者或使用者的私有环境一起放出去，公开仓库 / Release **不会包含**这些内容：

- 你自己的 `config.json`
- 运行历史
- 本地导出数据
- Cookie / Token / 密码 / 回调地址
- 本地调试临时文件
- 本机专属代理或面板配置

如果你下载后发现有些值是空的，这是正常的，因为这些本来就应该由你自己填。
