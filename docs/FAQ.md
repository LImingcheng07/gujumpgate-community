# FAQ

## 1. 这是官方版本吗？

不是。

这是一个 **community-maintained public derivative**，也就是社区整理维护的二开公开版，不代表上游作者的官方续作。

## 2. 这个项目现在还能稳定用吗？

不能做这种承诺。

这类项目高度依赖风控环境、代理质量、目标站策略和支付链路状态。今天能跑，不代表明天还能跑。

## 3. 现在推荐哪种导出方式？

当前更推荐：

- `导出至 - SESSION JSON 导入`

因为 OAuth 风控通常更重，很多时候会跳手机号绑定。

## 4. 仓库里为什么没有默认配置？

因为公开仓库不会附带个人环境配置。

例如这些内容都不会内置：

- `config.json`
- 真实邮箱
- 真实代理
- Token / Cookie / 回调地址
- 本地导出历史

你需要按自己的环境自行填写。

## 5. Release 压缩包和源码加载有什么区别？

区别不大，主要是获取方式不同：

- Release 压缩包：适合普通用户直接下载后解压加载
- 源码目录：适合要自己改代码、跟进更新、做二开的用户

## 6. 为什么我替换了文件但浏览器还是旧版本？

因为你大概率没有去 `chrome://extensions/` 页面点“重新加载”。

Chrome 不会在你替换本地文件后自动热更新扩展。

## 7. 我可以直接提 Issue / PR 吗？

可以。

但建议先：

1. 看 README
2. 看 Release 说明
3. 确认是不是环境问题
4. 提问题时尽量带上复现步骤、日志和截图

## 8. 为什么 README 里一直强调点 Star？

因为这个仓库本质上是学生维护、纯公益整理的公开版。

Star 不是钱，但对项目曝光、后续协作、维护动力都很重要。

## 9. 有打赏方式吗？

没有。

当前仓库是 **纯公益**，没有开放打赏入口。
